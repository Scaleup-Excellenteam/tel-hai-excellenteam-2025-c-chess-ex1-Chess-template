## תיאור האלגוריתם, ריבוי תהליכונים וסנכרון

### 1. סקירה כללית של האלגוריתם
- מנוע השחמט משתמש בשיטה פשוטה של **מינימקס ברוחב קבוע (Depth-2)** כדי להעריך מהלך אחד קדימה, ועד חמישה מהלכים בסך הכל (בעזרת הקריאה החוזרת למינימקס לעומק הנותר).
- בכל צעד, האלגוריתם בונה רשימה של **כל ההלכים החוקיים** האפשריים במצב הנוכחי, מבצע את כל אחד מהם לתוך עותק זמני של הלוח, מחשב `minimax(depth-1)` כדי להעריך את תוצאת המהלך, ואז בוחר את ההליכים בעלי הערך הגבוה ביותר.

### 2. חלוקת העבודה בין תהליכונים (Threads)
- כאשר המשתמש מבקש **ריבוי תהליכונים**, אנחנו יוצרים מטמון (`ThreadPool`) שמכיל מספר קבוע של *עובדים (workers)*.
- בכל קריאה ל־`recommendMoves(maxDepth, topN, pool)`, אנו מבצעים את הפעולות הבאות:
  1. יוצרים רשימה של כל מהלכי-מטרה אפשריים (זוגות src→dst לכל פיסה שיורדת בתור).
  2. לכל מהלך פותחים **משימה (job)** בבריכת התהליכונים:
    - בתוך המשימה, קוראים ל־`movePiece(...)` על עותק הלוח הזמני, ואז `minimax(maxDepth-1)`.
    - שומרים את תוצאת ההערכה במבנה משותף (עדיפות לפי ציון).
  3. כל משימה רצה במקביל, כך ש־N המהלכים הראשונים (או כל המהלכים במצב מסוים) מחולקים בין ה־workers.
  4. בסיום (או כאשר מתגלה מהלך מצוין ביותר ומסתיים מוקדם), אוספים את כל התוצאות ממחסנית התוצאות המשותפת.

- בניגוד להרצת מינימקס סדרתית, כאן כל מהלך מקבל Threads משלו לחישוב הערך שלו באותו ההקשר, דבר שמקצר משמעותית את זמן החיפוש לעומק קבוע (בעיקר במעבדים רב־ליבות).

### 3. סנכרון בין התהליכונים
- כדי להגן על גישה משותפת ל־`bestMoves` (ה־PriorityQueue ששומרת את עשרת המהלכים המובילים), השתמשנו ב־`std::mutex pqMutex`. כל כניסה/יציאה ל־PriorityQueue מתרחשת בלוק שמשתמש במנעול זה.
- קיימת גם דגל אטומי (`std::atomic<bool> stopFlag`) שמאפשר לממש Early-Exit: אם מוצאים מהלך ציון שנחשב “טוב מספיק” (למשל מעל סף מסוים), כל שאר המשימות תסיימנה מוקדם על-ידי בדיקה של `stopFlag.load()`.
- בעת שחזור מצב הלוח (העברת ה־sharedBoardString וה־whiteTurn למצב המקורי), אנחנו משתמשים במנעול נוסף (`restoreMutex`) כך ששתי משימות בו זמנית לא ישנו את הלוח יחד. כלומר, כל משימה שתנסה להריץ `movePiece(...)` תשמור תחילה עותק מקומי של `oldBoard` ו`oldTurn`, ותעדכן בחזרה בלוק באמצעות `restoreMutex`.
